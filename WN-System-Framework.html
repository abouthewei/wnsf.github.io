层级任务与核心职责

1. 前台层 (Front): 触达与交互

  • 职责: 负责与人（用户/员工/经销商）进行交互，接收指令并展示结果。
  • 特点: "只负责操作，不负责裁决"。所有复杂逻辑下沉至后台或中台。
  • 新增: 客服工作台 作为售后服务的统一入口；用户管理 包含C端App注册与B端账号管理。

2. 后台层 (Backend): 业务执行单元

  • 职责: 具体的业务逻辑执行者（如：生成一张订单、扣减一次库存、写一张NFC卡）。
  • 特点: "把事做完，上报事实"。每个后台系统专注于特定领域（Domain），产生不可篡改的原子事实数据。
  • 用户管理: 由 用户中心 (IAM) 负责身份/权限，CRM 负责商业关系/合同。
  • 经营增长域: 用户数据中心 (CDP) 负责行为标签与人群洞察，市场营销管理中心 (MA) 负责券/活动投放闭环，内容管理中心 (CM) 负责 AI 模板与内容资产管理。
  • 口径约束: CDP、MA、CM 负责执行与运营动作，其结果仍统一回流 数据中台，避免形成多套统计口径。

3. 中台层 (Middle): 裁决与数据资产

  • 数据中台: 唯一的"事实来源" (SSoT)。汇聚所有后台的流水，清洗为标准数据，并对外提供查询 API。
  • 分账中台: 唯一的"规则裁决者"。基于数据中台的事实，计算该给谁分多少钱，并指令财务系统打款。

关键流转总结

1
触发 (前台 -> 后台)
用户下单、领券/参与活动、调用内容模板，客服发起退款、运维提交工单。
2
执行与上报 (后台 -> 数据中台)
订单系统生成订单，支付系统扣款，营销系统发券，内容中心记录模板调用，CDP 沉淀行为标签。完成后，作为事实(Facts)异步上报给数据中台。
3
裁决 (数据中台 <-> 分账中台)
中台基于上报的事实，计算分账规则，生成应收应付清单。
4
资金执行 (中台 -> 财务系统)
分账中台指令财务执行系统进行打款/开票。
5
反馈 (API 服务)
前台通过调用数据中台 API 读取最终状态与报表。
