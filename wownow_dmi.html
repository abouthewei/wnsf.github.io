WOWNOW DMI 方案讨论

WOWNOW DMI 方案讨论页

基于「Admin 统一配置 + 商务代理创建」模式

DMI 不是优惠券、团购或普通订单系统，而是 WOWNOW 从"单用户定制"升级为"组织型批量发行"的基础能力。第一版不做完整 B 端 SaaS，而是先跑通核心闭环。

组织型批量发行

DMI，Digital Mass Issuance，是 WOWNOW 面向个人、企业、学校、品牌、IP、渠道和自营场景的 TimeTag 实体数字资产批量发行系统。通过 Admin 统一配置、商务代理创建，先跑通发行、领取、创作、下单、资产、NFC、生产、履约、数据和分账记录的核心闭环。

DMI 是什么

它不是

  • 不是优惠券系统
  • 不是普通团购系统
  • 不是普通订单系统
  • 不是单纯 NFC 管理系统

它是

  • 一个 DMI 项目
  • 多个领取用户
  • 多个下单权益
  • 多个个性化作品
  • 多个 TimeTag 资产
  • 有限 TimeForge 设备生产
  • 持续 NFC 内容运营
  • 多主体收益分账

核心结论

  • 从"一个用户下一个定制订单"
  • 升级为"一个组织发起一次发行"
  • 多个用户自主领取、创作、下单
  • 生产和绑定资产
  • 这是 WOWNOW 的关键基础设施
DMI 是 WOWNOW 从"单用户定制"升级为"组织型批量发行"的关键基础设施。

为什么 WOWNOW 现在需要 DMI

当前问题

  • 批量项目难承接 — 企业、学校、品牌活动来了，只能人工收集、人工处理
  • 个性化和批量冲突 — 组织方不可能替每个用户收集照片、文案、内容
  • 生产履约不闭环 — 领取、创作、下单、生产、配送、售后无法系统化管理
  • TimeForge 产能难调度 — 设备有限，但订单可能来自自营、渠道、企业、个人
  • B 端数据不可沉淀 — 客户不知道谁领取、谁创作、谁生产、谁触碰 NFC
  • 渠道收益不好定义 — 渠道可能同时带客户、持有设备、负责履约
  • NFC 后续运营断裂 — 实物发出后，内容、权益、触碰数据没有统一运营入口

DMI 价值

  • 对 WOWNOW — 从单个 AI 定制订单升级为组织型发行平台
  • 对企业 / 学校 / 组织 — 降低收集、校对、设计、分发、售后成本
  • 对用户 — 自己领取、自己创作、自己绑定，获得专属 TimeTag
  • 对渠道 — 可以通过销售、设备、履约获得收益
  • 对 IP / 品牌 — 将周边变成可追踪、可运营的实体数字资产
  • 对 TimeForge — 让有限设备产能进入可调度、可结算的商业体系
DMI 是承接企业、学校、品牌、IP 与渠道型项目的必要基础设施，否则批量项目只能依赖人工交付，无法规模化。

DMI 涉及的角色关系

DMI 的角色必须拆开，不能只按"谁创建项目"来判断收益、责任和权限。

可以发起 DMI 的角色

  • 个人用户
  • 企业 / 组织 / 机构
  • IP / 品牌方
  • WOWNOW 自营
  • WOWNOW 渠道运营商

业务主体

  • Creator 创建人 — 谁在系统里创建项目
  • Issuer 发行方 — 项目名义上的发行主体
  • Customer 客户主体 — 谁采购、付款、签合同
  • Seller 销售方 — 谁带来客户或订单
  • Channel 渠道方 — 是否来自渠道运营商
  • Platform 平台方 — WOWNOW
  • Manufacturer 生产方 — 谁的设备实际生产
  • Fulfillment 履约方 — 谁负责交付和售后
  • IP Owner 内容方 — 谁提供 IP、音乐、品牌、素材、授权

重点不等式

  • 创建人 ≠ 发行方
  • 发行方 ≠ 销售方
  • 销售方 ≠ 生产方
  • 生产方 ≠ 履约方

TimeTag 资产与权利归属

DMI 用户下单后，TimeTag 默认进入用户的"我的 TimeTag 资产"，但必须标注资产类型和权利边界；进入用户资产 ≠ 用户拥有全部作品版权。

DMI 下 TimeTag 包含

TimeTag 实物用户或组织创作的作品NFC 绑定的数字内容 / 权益后续可运营、可补发、可加购的资产记录

必须区分的权利

权利类型含义示例
实体所有权实物 TimeTag 归谁用户拥有实体 Tag
作品版权图案、音乐、文字、设计版权归谁用户原创 / IP 方 / 组织方 / 平台模板
数字内容管理权NFC 打开后的内容谁能编辑用户可编辑 / 组织可更新 / 平台维护
IP 授权使用权用户是否只是被授权收藏或使用粉丝卡、品牌联名款
NFC 权益访问权是否拥有门票、会员、优惠等权益门票、会员卡、隐藏内容
补发重制权益遗失后能否重新制作免费补发 / 自费补发 / 不支持
转赠权用户能否把资产转给别人P0 不做，P1 后按资产类型配置
商业使用权用户能否商用图案或内容默认不可商用

资产类型标签

原创资产授权资产权益资产纪念资产平台模板资产临时资产组织资产

资产与实体 Tag 的关系

TimeTag Asset ├── Physical Tag 1：旧 Tag，正常 / 挂失 / 作废 └── Physical Tag 2：补发新 Tag，正常
用户遗失的是实体 Physical Tag，不一定丢失 TimeTag Asset。补发时，原 TimeTag Asset 保留，旧 Physical Tag 冻结 / 作废，新 Physical Tag 绑定到原资产，NFC 内容可继承原资产内容。

TimeForge 设备与利益关系

DMI 发起权可以开放，但 TimeForge 产能权是有限主体持有的。项目可以由不同角色发起，但生产收益必须按实际设备归属结算。

收益拆分原则

  • WOWNOW 提供平台、账户、资产、NFC、数据 → WOWNOW 固定收平台服务费
  • 谁带来客户 / 订单 → 谁拿销售收益
  • 谁的设备实际生产 → 谁拿生产收益
  • 谁负责交付和售后 → 谁拿履约收益
  • 谁提供 IP / 内容授权 → 谁拿授权收益

典型场景

  • 自营 + 自营设备 → WOWNOW 拿平台 + 销售 + 生产 + 履约收益
  • 自营 + 渠道设备 → WOWNOW 拿平台 + 销售，渠道拿生产 / 履约
  • 渠道项目 + 渠道设备 → 渠道拿销售 + 生产 + 履约，WOWNOW 拿平台费
  • 渠道项目 + 自营设备 → 渠道拿销售，WOWNOW 拿平台 + 生产
  • IP + 渠道 + 渠道设备 → IP 拿授权，渠道拿销售 / 生产 / 履约，WOWNOW 拿平台费
核心原则：谁实际生产，谁获得设备生产收益；WOWNOW 始终保留平台服务收益。

第一层：Admin 代理创建

P0 目标 · Admin 统一配置 · 商务代理创建 · 先跑通核心闭环

为什么先做代理创建

  • 规则复杂，容易配错
  • 商务属性强，需要报价、合同
  • 早期要控风险，版权、内容、资产需把关
  • 研发成本低，先不做完整 B 端 SaaS
  • 更适合验证，用商务代建先跑通闭环

当前阶段不做

  • 企业自助创建后台
  • IP / 品牌自助创建
  • 完整渠道后台

角色创建方式

  • 个人用户 → C 端轻量创建
  • WOWNOW 自营 → Admin 创建
  • 商务 / 运营 → Admin 代理创建
  • 渠道运营商 → Admin 受限账号
  • 企业 / 组织 → 商务代理创建
  • IP / 品牌方 → 商务代理创建

风险管控

DMI 涉及多方主体、资产权利和资金流转，必须提前识别风险并制定应对策略。

版权与 IP 风险
  • 风险：用户上传侵权图片、使用未授权 IP、生成内容涉及版权争议
  • 应对：P0 通过项目配置限制创作范围 + 人工审核；P1 引入自动审核；P2 建立 IP 授权库
  • 底线：任何 TimeTag 资产必须标注版权归属，用户创作默认保留版权但授予平台展示和制作授权
资产权与纠纷风险
  • 风险：用户认为"我的资产"等于"我拥有全部权利"，实体遗失后要求赔偿或争议
  • 应对：领取和下单时明确展示资产类型和权利边界；资产详情页标注权利范围
  • 底线：实体所有权归用户，但作品版权、IP 授权、数字内容管理权按项目配置归属
分账与资金风险
  • 风险：多主体分账计算错误、渠道收益争议、自动打款出错
  • 应对：P0 只做分账记录不做自动打款；P1 结算报表 + 人工确认打款；P2 自动分账
  • 底线：所有分账规则在项目创建时锁定，变更需记录操作日志
生产与履约风险
  • 风险：TimeForge 设备故障导致批量延期、质检不合格、NFC 写入失败
  • 应对：指定设备 + 简单派单；预留异常上报和重做流程；生产状态实时同步
  • 底线：每个生产订单必须有质检记录，NFC 写入必须验证成功
用户隐私与数据风险
  • 风险：用户照片、纪念语等个人信息泄露；企业员工数据被不当使用
  • 应对：数据按项目隔离；用户内容仅项目相关方可见；NFC 内容用户可控
  • 底线：遵守数据保护法规，用户可删除自己的创作内容（删除内容不影响已生产实体）
运营与内容风险
  • 风险：NFC 内容被篡改、违规内容通过审核、恶意领取占用名额
  • 应对：P0 人工审核 + 白名单领取；P1 自动审核 + 黑名单；P2 内容安全体系
  • 底线：NFC 内容变更需审核，恶意用户可封禁

  • 用户使用 AI 创作的作品，版权归谁？用户 / 平台 / 共有？

  • 是否需要在领取 / 下单时展示版权声明？
分账规则是否在项目创建时锁定？
  • 项目进行中能否修改分账规则？
  • 如果修改，已产生的订单按新规则还是旧规则？
  • 是否需要分账规则版本管理？

  • P0 指定设备生产，是否足够？

  • 设备故障时如何自动切换？
遗失补发的收费策略？
  • 补发是否收费？按什么标准？

渠道接入的边界在哪里？
  • 渠道可以做什么？只能带客户，还是也能持有设备、负责履约？

NFC 内容运营的边界？
  • NFC 内容由谁运营？用户 / 发行方 / 平台？

  • NFC 内容变更是否需要用户确认？
DMI 与现有 C 端定制的关系？

  • DMI 用户后续能否直接使用 C 端定制功能？

最终建议

DMI 是 WOWNOW 从"单用户定制"升级为"组织型批量发行"的关键基础设施。第一版不做完整 B 端 SaaS，而是先跑通核心闭环，用真实项目验证模式。

P0 先跑通

  • Admin 统一配置
  • 商务代理创建
  • 3 个真实项目验证
  • 核心闭环跑通

风控先行

  • 版权标注必须
  • 资产权利明确
  • 分账记录先行
  • 操作日志完整

渐进开放
